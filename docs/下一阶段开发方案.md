### 2. 订阅生命周期 Worker

#### 背景
当前订阅服务只有状态查询和手动调整，缺少自动化生命周期管理（试用到期、宽限到期、暂停、保留期清理）。

#### 实施步骤

**2.1 生命周期处理逻辑**
```python
# workers/subscription_worker.py
class SubscriptionLifecycleWorker:
    async def process_lifecycle_events(self):
        # 试用到期 → 检查支付 → 转活跃/宽限
        # 宽限到期 → 暂停
        # 保留期到期 → 标记待清理
        # 续费订单到期 → 取消
```

**2.2 关键场景**
- 试用期结束：检查是否有付款，未付款进入宽限期
- 当前周期结束：检查续费订单，未付款进入宽限期
- 宽限期结束：暂停租户，清理 Webhook 缓存
- 保留期结束：标记数据待清理（不实际删除）
- 自动续费失败重试（如有支付配置）

**2.3 通知机制**
- Telegram 消息通知租户 owner（试用到期、宽限、暂停）
- 审计日志记录所有状态变更
- 平台管理员可在 Admin Web 查看关注队列

**2.4 测试覆盖**
- 各生命周期阶段转换测试
- 时间边界测试（提前1小时、当天、延迟处理）
- 并发安全测试（多 Worker 实例）

#### 工作量估算
- 生命周期逻辑: 1.5天
- Telegram 通知: 0.5天
- 测试和调试: 1天
- **合计: 3天**
