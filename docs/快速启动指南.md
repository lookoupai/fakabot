# 🚀 FakaBot 快速启动指南（Docker Compose 部署）

> 推荐 Docker 部署。数据库、Redis、应用一并编排，启动时自动执行数据库迁移。

## 前置条件

- Docker 与 Docker Compose（compose v2）
- 对外可访问的域名 + HTTPS（用于 Telegram Webhook 回调）

---

## 步骤 1：准备环境配置

```bash
cp .env.example .env
```

编辑 `.env`，**至少**确认/修改以下项：

| 配置项 | 说明 |
|---|---|
| `PUBLIC_BASE_URL` | 你的公网域名（Webhook 注册地址），如 `https://fakabot.example.com` |
| `TOKEN_ENCRYPTION_KEY` | 租户配置加密密钥，**必填**，`openssl rand -hex 32` 生成 |
| `MASTER_WEBHOOK_SECRET` | 母 Bot Webhook 密钥，随机字符串 |
| `PLATFORM_ADMIN_IDS` | 你的 Telegram 数字用户 ID（逗号分隔），用于平台后台登录 |
| `MASTER_BOT_TOKEN` | 母 Bot Token（仅系统通知用，可选） |

> 数据库与 Redis 连接在 docker 部署下使用容器内部地址（`postgres:5432` / `redis:6379`），`.env` 里默认值已就绪，无需修改。

## 步骤 2：启动服务

```bash
docker compose up -d --build
```

这会启动三个容器：

| 容器 | 作用 |
|---|---|
| `fakabot-postgres` | PostgreSQL 15 |
| `fakabot-redis` | Redis 7 |
| `fakabot` | 应用 + 内置后台 Worker（`app` 容器启动时通过 `docker-entrypoint.sh` 自动执行 `alembic upgrade head`） |

> 应用进程内置了 `BackgroundWorkerManager`，会自动运行订单过期、库存解锁、支付对账、**支付回调重试**、外部发货、发货派送、账单结算、报表导出、订阅生命周期等后台任务，无需单独启动 Worker 进程。

## 步骤 3：验证

```bash
# 容器状态（应均为 healthy）
docker compose ps

# 存活探针（仅检查进程/连接）
curl http://127.0.0.1:58001/health

# 就绪探针（检查 DB 连接 + 核心表已迁移 + Redis + 后台任务存活）
curl http://127.0.0.1:58001/ready

# 详细健康（各组件 + 后台任务列表）
curl http://127.0.0.1:58001/health/detailed
```

> `/ready` 会校验核心业务表已迁移就绪，避免「数据库为空却报 healthy」的假阳性。

## 步骤 4：访问管理后台

浏览器打开（默认监听 `127.0.0.1`，需经反向代理暴露到公网域名）：

```
http://127.0.0.1:58001/admin
```

通过 Telegram 登录或一次性绑定码进入租户/平台管理界面。

---

## 端口与反向代理

- 容器内应用监听 `58001`，`docker-compose.yml` 映射为 `127.0.0.1:58001`（仅本机）。
- 公网访问请在宿主机用 Nginx/Caddy 反代 `127.0.0.1:58001` 到你的域名，并配置 HTTPS。

## 支付 / 货源配置

支付网关（EPUSDT、易支付）与外部货源（mcy_shop、standard_http）是**租户级配置**，在 Admin Web 中按租户配置，**无需在 `.env` 中设置**。

## 常见问题

**Q：容器显示 healthy，但接口报错？**
A：早期版本 `/ready` 只检查连接不检查表，可能假阳性。现已修复；若仍异常，看 `docker compose logs fakabot` 是否有 `UndefinedTableError`（迁移未执行）。

**Q：数据库迁移什么时候执行？**
A：`fakabot` 容器每次启动都会先跑 `alembic upgrade head`（见 `docker-entrypoint.sh`），重建/重启容器即自动迁移。

**Q：如何重新生成 `TOKEN_ENCRYPTION_KEY`？**
A：`openssl rand -hex 32`。注意：更换后历史加密配置无法解密，需重新录入。

**Q：如何启动 USDT 链上扫描器（多 Bot 场景）？**
A：`docker compose --profile multi-bot up -d`，会额外启动 `usdt_scanner` 服务。

## 非 Docker 部署（备选）

```bash
# 1. 准备 PostgreSQL / Redis 并写入 .env
# 2. 安装依赖
python -m venv .venv && . .venv/bin/activate && pip install -r requirements.txt
# 3. 迁移
alembic upgrade head
# 4. 启动
uvicorn app.main:app --host 0.0.0.0 --port 58001
```

后台 Worker 已内置在应用进程中，无需单独运行 `workers/*`。
