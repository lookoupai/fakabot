# 🚀 FakaBot 快速启动指南

## 步骤1: 复制环境配置

```bash
cp .env.example .env
```

## 步骤2: 编辑 .env 文件（必填项）

### 必须配置的项目

```bash
# 数据库连接（必需）
DATABASE_URL=postgresql+asyncpg://用户名:密码@主机:端口/数据库名
# 例如: postgresql+asyncpg://fakabot:your_password@localhost:5432/fakabot

# Redis 连接（必需）
REDIS_URL=redis://主机:端口/数据库号
# 例如: redis://localhost:6379/0

# 公网访问地址（必需，用于 Webhook 等）
PUBLIC_BASE_URL=https://你的域名.com

# 存储目录（必需）
STORAGE_ROOT=/www/wwwroot/fakabot/storage

# 密钥加密（必需，用于加密租户配置）
TOKEN_ENCRYPTION_KEY=生成一个随机字符串
# 例如: openssl rand -hex 32
```

### 可选配置（看需要）

```bash
# Telegram 母 Bot（如果要用 Telegram 通知功能）
MASTER_BOT_TOKEN=你的Bot Token
MASTER_WEBHOOK_SECRET=随机字符串

# 平台管理员（可选，用于访问平台管理后台）
PLATFORM_ADMIN_IDS=你的Telegram用户ID

# EPUSDT 支付（可选，也可以在 Admin Web 中配置）
EPUSDT_BASE_URL=
EPUSDT_PID=
EPUSDT_SECRET_KEY=
```

## 步骤3: 创建数据库和存储目录

```bash
# 创建数据库（如果还没有）
createdb fakabot

# 创建存储目录
mkdir -p /www/wwwroot/fakabot/storage
```

## 步骤4: 执行数据库迁移

```bash
cd /www/wwwroot/fakabot
alembic upgrade head
```

## 步骤5: 启动服务

### 方式1: 使用启动脚本（推荐）

```bash
# 启动 Web 服务
./scripts/start_service.sh

# 或者启动 Web + Workers
START_WORKERS=true ./scripts/start_service.sh
```

### 方式2: 手动启动

```bash
# 启动 Web 服务
.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000

# 启动 Worker（另开终端）
.venv/bin/python -m workers.report_worker
.venv/bin/python -m workers.subscription_worker
.venv/bin/python -m workers.payment_retry_worker
```

## 步骤6: 访问管理后台

浏览器打开：
```
http://localhost:8000/admin
```

或者你配置的域名：
```
https://你的域名.com/admin
```

## 🔧 配置说明

### 关于支付和外部货源配置

**重要**: 支付网关和外部货源**不需要在 .env 中配置**！

它们是**租户级配置**，每个租户通过 Admin Web 界面配置自己的：
- EPUSDT 配置（商户ID、密钥）
- 易支付配置（PID、密钥）
- 外部货源配置（mcy_shop、standard_http）

### 关于 Telegram 通知

- `MASTER_BOT_TOKEN`：只用于发送系统通知（订阅到期提醒等）
- 每个租户的克隆 Bot 有自己独立的 Token
- 通过 Admin Web 界面管理

## 🐛 常见问题

### Q: 数据库连接失败？
A: 检查 `DATABASE_URL` 是否正确，数据库服务是否启动

### Q: Redis 连接失败？
A: 检查 `REDIS_URL` 是否正确，Redis 服务是否启动

### Q: Telegram 通知不发送？
A: 检查 `MASTER_BOT_TOKEN` 是否配置，Worker 是否启动

### Q: 如何生成 TOKEN_ENCRYPTION_KEY？
A: 运行 `openssl rand -hex 32` 生成随机密钥
