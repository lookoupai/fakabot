## 🎨 功能详解

### 1. 报表生成 Worker ✅

**支持的报表类型**:
- `orders` - 订单报表
- `payments` - 支付记录
- `inventory` - 库存统计
- `ledger` - 账本流水
- **`products`** - 商品报表（新增）

**商品报表字段**:
```csv
商品ID,商品名称,分类,排序,状态,发货类型,价格,币种,可用库存,创建时间,更新时间
```

**安全边界**:
- ❌ 不导出库存明文内容
- ❌ 不导出文件 storage key
- ❌ 不导出外部商品映射
- ❌ 不导出供应商/代理商信息

**运行参数**:
- 处理间隔: 30秒
- 每批处理: 5个任务
- 流式写入大数据量

---

### 2. 订阅生命周期 Worker ✅

**状态转换**:
```
trial → active (有付款) / grace (无付款)
      ↓
active → active (已续费) / grace (未续费)
       ↓
grace → active (补付款) / suspended (宽限到期)
      ↓
suspended → (数据保留期 30天)
          ↓
      标记待清理（不删除数据）
```

**处理逻辑**:
- ✅ 试用期到期检查
- ✅ 当前周期结束检查
- ✅ 宽限期到期检查
- ✅ 保留期到期检查
- ✅ 完整审计日志记录
- ⚠️ Telegram 通知（TODO）

**运行参数**:
- 处理间隔: 1小时
- 每批处理: 100个订阅
- 并发安全（行锁）

---

### 3. 支付回调重试 Worker ✅

**重试策略**:
```
第1次重试: 1分钟后
第2次重试: 5分钟后
第3次重试: 30分钟后
超过3次: 不再重试
超过24小时: 自动放弃
```

**运行参数**:
- 处理间隔: 5分钟
- 每批处理: 20个回调
- 指数退避避免过度重试

**Q: 商品报表如何创建？**  
A: 通过 Admin Web 报表页面选择 "products" 类型

**Q: Worker 如何监控？**  
A: 查看日志 `journalctl -u fakabot-*-worker -f`，检查心跳 `redis-cli GET worker:*:heartbeat`

**Q: 订阅状态何时自动转换？**
