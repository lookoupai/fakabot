### 3. 服务启动脚本 ✅

**文件**: `scripts/start_service.sh`

**功能**:
- ✅ 环境检查（.env、虚拟环境）
- ✅ 配置验证（必需环境变量）
- ✅ 数据库连接测试
- ✅ Web 服务启动（Uvicorn）
- ✅ Worker 进程启动（可选）

**使用方法**:
```bash
# 仅启动 Web 服务（默认）
./scripts/start_service.sh

# 启动 Web + Workers
START_WORKERS=true ./scripts/start_service.sh

# 自定义端口和工作进程数
WEB_PORT=9000 WEB_WORKERS=8 ./scripts/start_service.sh
```

**启动流程**:
```
[1/5] 检查环境
  - 验证 .env 文件存在
  - 验证虚拟环境存在

[2/5] 验证配置
  - 检查 DATABASE_URL
  - 检查 SECRET_KEY
  - 检查 STORAGE_ROOT

[3/5] 测试数据库连接
  - 执行 SELECT 1 测试

[4/5] 启动服务
  - 显示启动配置

[5/5] 执行启动
  - Web: uvicorn app.main:app
  - Workers: 后台进程
```

**环境变量**:
- `START_WEB` - 是否启动 Web（默认: true）
- `START_WORKERS` - 是否启动 Worker（默认: false）
- `WEB_PORT` - Web 端口（默认: 8000）
- `WEB_WORKERS` - 工作进程数（默认: 4）

## 技术亮点

### 1. Worker 心跳监控

```python
# 检查 Worker 心跳
heartbeat_key = f"worker:{worker_name}:heartbeat"
heartbeat = await redis_client.get(heartbeat_key)

# 解析心跳时间
heartbeat_time = datetime.fromisoformat(heartbeat)
age_seconds = (datetime.now(timezone.utc) - heartbeat_time).total_seconds()

# 判断健康状态
if age_seconds < 300:  # 5分钟内
    status = "healthy"
else:
    status = "stale"
```

### 2. 详细健康检查

返回所有组件的详细状态，便于排查问题：
- 每个组件独立检查
- 异常时不中断其他检查
- 返回完整错误信息

启动前逐步验证：
1. 文件存在性
2. 环境变量完整性
3. 数据库连接
4. 然后才启动服务
