**任务：构建一个 Telegram 多租户 Bot 克隆平台（Bot-as-a-Service / SaaS）**

### 项目目标
开发一个 Telegram Bot 克隆系统，用户可以通过一个**母 Bot** 轻松克隆出属于自己的独立 Bot，而无需自己部署服务器。

核心特点：
- 用户只需提供自己通过 @BotFather 创建的 **Bot Token**，即可获得一个功能完整、**可个性化设置**的独立 Bot。
- 所有克隆 Bot 共享同一套后端代码和服务器，但每个 Bot 的数据和设置完全隔离。

### 技术栈要求
- 语言：**Python**（推荐使用 **aiogram 3.x**）
- 模式：**Webhook**（必须支持多 Token）
- 数据库：PostgreSQL + Redis（缓存配置）
- 架构：**单进程多租户**（不要为每个用户单独启动实例）

### 核心功能要求
1. **母 Bot（Cloner Bot）**：
   - 引导用户输入 Bot Token
   - 自动为用户 Token 设置 Webhook（指向本服务器）
   - 支持管理已克隆的 Bot（查看状态、删除、重置 Webhook）

2. **多租户机制**（最重要）：
   - 根据 incoming Update 中的 Token 识别所属 Bot
   - 所有设置、用户数据、功能配置都按 `bot_token` 隔离
   - 使用 Middleware 动态加载当前 Bot 的配置

3. **个性化设置系统**：
   - 每个克隆 Bot 可独立设置自己的欢迎语、菜单、功能开关、自定义参数等
   - 支持命令式设置（如 `/setwelcome`、`/addbutton` 等）

4. **通用扩展性**：
   - 方便后续添加具体业务功能（例如：消息转发、AI 对话、群组管理、文件下载、投票等）
